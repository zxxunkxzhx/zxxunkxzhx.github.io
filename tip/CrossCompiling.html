MSVC

As is known that MSVC is microsoft "visual" toolchain to develop c(++) apps.

To cross-compile from x64 to arm64, we simply call "Microsoft Visual Studio\...\Build\vcvarsall.bat" x64_arm64

To use build system such as CMake, you call `cmake -G "..." -A ARM64`; pupular CMAKE_VS_PLATFORM_NAME are "ARM64","Win32"(not "x86"!) and "x64".

GNU C(++) toolchains

Unlike MSVC, you have to provide a specific tool chain to cross-compile.

When cross-compiling, CMAKE_TOOLCHAIN_FILE should set the CMAKE_SYSTEM_PROCESSOR variable to match target architecture that it specifies.

For MSVC, CMAKE_SYSTEM_NAME is always "Windows", yet for gcc toolchains, it can (also) be: Android, Darwin, iOS, Linux, etc; note although Android is derived from Linux, we set aside it

clang toolchains

Almost all GNU compile options are accepted by clang(LLVM compilier).

If your target arch is apple(such as iMac), you involves clang; clang is tightly integrated with Xcode and supports Apple-specific toolchains like Objective-C and Swift.
