rule

A rule begins with target(s); A goal is the target (or set of targets) that make is asked to build

When a rule has multiple targets, it is equivalent to writing independent rules for each target.

Rule lists some files that the targets depend on, called the prerequisites; such as

$(BUILDDIR)/%.o: $(SRCDIR)/%.c
    ;

Rule may also give a recipe to use to create or update the targets.

variable definition

A variable definition is a line that specifies a text string value for a variable.

directive

  • `include filename1 [filename2...]` or `-include filenames...`(sinclude, silent include)
  • `ifeq ($(CC),gcc)`, ifdef $(foo), ...
  • Defining Multi-Line Variables, such as 
    define two-lines
echo foo
echo $(bar)
endef

#

  • within variable references: treated literally, such as $(my#var)
  • within function call: treated literally, such as BAR = $(subst a, b#, foo)
  • within a recipe: whether or not this is a comment is up to the shell
  • within a define directive: either be treated as make comments or as recipe text, depending on the context in which the variable is evaluated
