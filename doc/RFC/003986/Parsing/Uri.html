scheme

scheme = ALPHA *( ALPHA / DIGIT / "+" / "-" / "." ) where ALPHA is caseless; if a buf does not starts with a scheme, it's not a URI(but may be a relative-ref)

branches of hier-part

  1. "//" authority path-abempty: if a URL's hier-part begins with "//", an authority(may be an empty authority) follow it(path-absolute begins with a '/' and a pchar, path-rootless begins with a pchar)
  2. path-absolute: if scheme is "data"(zxxu: archive file specifying data in it), in theory we can figure out a URI which has its hier-part as path-absolute(note file:///path/to/your/file belong to the first branch which has authority as empty)
  3. path-rootless: some URI don't have a concept of path or root-path
  4. path-empty: such as "mailto:" is valid and will open a blank email client

authority( in hier-part 01 )

authority = [ userinfo "@" ] host [ ":" port ]
  • userinfo = *( unreserved / pct-encoded / sub-delims / ":" ) ':' is a gen-delim, but it's safe to occure here
  • IPv4address (type host) begins with a dec-octet
  • IP-literal (type host) begins with '[' and followed by IPv6address or IPvFuture; Note '[' is a gen-delim which only used for this purpose
  • reg-name (type host) is *( unreserved / pct-encoded / sub-delims ); first-match-wins: If host matches the rule for IPv4address, then it should be considered an IPv4 host and not a reg-name.
