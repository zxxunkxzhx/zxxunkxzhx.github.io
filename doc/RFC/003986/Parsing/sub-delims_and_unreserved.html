sub-delims are used by specific schemes; not considering percent-encoding, a char in URI is an unreserved char if it's neither gen-delim nor sub-delim.

unreserved = ALPHA / DIGIT / "-" / "." / "_" / "˜"

besides delims and unreserved, there are chars which can't be in URI, such as the ^(%5E) char

to better remember sub-delim char, we list their use cases( in diff schemes ):

  • in the Java jar: URL scheme, ! separates the outer JAR file from the inner resource.
  • In HLS media playlists, $ encloses HLS variables
  • & separates key-value pairs in a HTTP(S) query string
  • XRI 2.0 (an OASIS standard) treated ' as a delimiter before local access identifiers.
  • In mailto:user(name)@example.com , ( and ) carry meaning (they start and end a comment).
  • S3 paths support * as a wildcard
  • RFC 3966 allows + as a global-dialing prefix marker
  • RFC 6068 allows comma-separated lists of recipients: mailto:alice@example.com,bob@example.com
  • FTP URLs allow ; in FTP parameter extensions: ftp://host/path;type=a
