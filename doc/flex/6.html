`.` matches any character(byte) except newline(\n)

The {-} syntax in flex represents set subtraction, such as [a-z]{-}[aeiou]

\a means the bell char

r$ means a pattern r but only at the end of a line. Equivalent to r/\n(for DOS file, use r/\r\n instead of r$; or we can use r$ on any plat but filter out the last \r char which is matched)
