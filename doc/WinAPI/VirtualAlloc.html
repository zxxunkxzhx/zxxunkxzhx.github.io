Return

The base address of the allocated region of pages.

`LPVOID lpAddress` arg

The starting address of the region to allocate.

If the memory is being reserved, the specified address is rounded down to the nearest multiple of the allocation granularity; To determine the size of a page and the allocation granularity on the host computer, use the GetSystemInfo function.

If the memory (is already reserved and) is being committed, the address is rounded down to the next page boundary(zxxu: current page's page boundary?).

`SIZE_T dwSize` arg

The size of the region, in bytes. If the lpAddress parameter is NULL, this value is rounded up to the next page boundary. Otherwise, the allocated pages include all pages containing one or more bytes in the range from lpAddress to lpAddress+dwSize.

flAllocationType.MEM_COMMIT

Allocates memory charges (from the overall size of memory and the paging files on disk) for the specified reserved memory pages. The function also guarantees that when the caller later initially accesses the memory, the contents will be zero(zxxu: re-commit don't make sure all zeros)

flAllocationType.MEM_RESERVE

memory returned by MEM_RESERVE can be MEM_RELEASE in VirtualFree.

We can simultaneously reserve and commit a region of free pages

normally 0
