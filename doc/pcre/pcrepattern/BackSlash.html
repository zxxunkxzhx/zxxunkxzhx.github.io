  • [recognized both inside and outside character classes]If you want to remove the special meaning from a sequence of characters, you can do so by putting them between \Q and \E
    An isolated \E that is not preceded by \Q is ignored, such as the pattern `\E` matches exactly "E"
    If \Q is not followed by \E later in the pattern, the literal interpretation continues to the end of the pattern.
    If the isolated \Q is inside a character class, this causes an error, because the character class is not terminated.

  • back references: The sequence \g followed by an unsigned or a negative number, optionally enclosed in `{}`, is an absolute or relative back reference. A named back reference can be coded as \g{name}. Such as `(a)(b)\g{2}c` against subject "abbc".
  • Absolute and relative subroutine calls: For compatibility with Oniguruma, \g followed by a name or a number enclosed either in angle brackets or single quotes, is an alternative syntax for referencing a subpattern as a "subroutine".

  • Resetting the match start
    The escape sequence \K causes any previously matched characters not to be included in the final matched sequence. For example, the pattern foo\Kbar matches "foobar", but reports that it has matched at offset 3. \K is acted upon when it occurs inside positive assertions, but is ignored in negative assertions. Note that when a pattern matches, the reported start of the match can be greater than the end of the match. Such as when pattern (?=ab\K) performed on xabyz, group 0 is reported to be matched as [3,1]
